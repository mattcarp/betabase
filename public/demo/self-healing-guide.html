Self-Healing Test Dashboard

Demo Guide - Test Pillar of SIAM's Three-Pillar Architecture

Overview

The Self-Healing Test Dashboard demonstrates AI-powered test maintenance that automatically repairs broken UI tests when selectors change. This is the Test pillar of SIAM's three-pillar architecture (Chat, Curate, Test).

Three-Tier Healing System

The system classifies each broken test into one of three tiers based on AI confidence:

TIER 1
Automatic Healing
Confidence: >90% - Heals automatically, no human intervention needed.
Example: Button text changed from "Submit" to "Send" - obvious match.
TIER 2
Review Required
Confidence: 60-90% - Queued for QA review with AI recommendation.
Example: Multiple possible matches, needs human judgment.
TIER 3
Architect Review
Confidence: <60% - Escalated to test architect.
Example: Component completely restructured, may need test rewrite.

Code Diff Visualization

When reviewing a healed test, the dashboard displays the selector change:

- button[data-testid="submit-btn"] + button[data-testid="send-btn"]

The diff shows the original selector (red) and the AI-suggested replacement (green), along with confidence score and DOM context.

Cascade Healing

One fix can repair multiple tests.

When the AI fixes a selector pattern, it automatically identifies and repairs all other tests using the same pattern. This dramatically reduces maintenance burden.

Demo talking point: "When we fix one test, the system automatically identifies 12 other tests with the same selector pattern and heals them all. That's cascade healing."
